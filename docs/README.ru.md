# Плагин Push-уведомлений для osTicket

Web Push (PWA) уведомления для панели сотрудников osTicket. Доставляет push-уведомления браузера в реальном времени для событий, связанных с заявками, полностью независимо от уведомлений по электронной почте.

## Возможности

- **Push-уведомления в реальном времени** для: новых заявок, новых сообщений/ответов, назначений, передач, просроченных заявок
- **Независимость от email-уведомлений** — работает даже при полностью отключённых уведомлениях по электронной почте
- **Настройки агента** с переключателями по типу событий, фильтрацией по отделам и режимом тишины
- **Управление администратором** с главным переключателем, переключателями по типу событий, пользовательской иконкой уведомлений и управлением ключами VAPID
- **Поддержка нескольких языков** с использованием встроенной системы переводов osTicket
- **Адаптивный интерфейс для мобильных устройств** с иконками колокольчика и шестерёнки в мобильной навигационной панели
- **Совместимость с тёмной темой** (тема osTicketAwesome)
- **Основан на Service Worker** — работает даже при закрытой вкладке браузера
- **Нет зависимостей** — чистая PHP-реализация Web Push, не требует Composer

## Требования

- osTicket **1.18+**
- PHP **8.0+** с расширением `openssl` (для генерации ключей VAPID и шифрования данных)
- HTTPS (требуется Web Push API)

## Установка

1. Скопируйте папку `push-notifications/` в `include/plugins/`
2. В панели администратора перейдите в **Управление > Плагины > Добавить новый плагин**
3. Нажмите **Установить** рядом с «Push-уведомления»
4. Установите статус **Активный** и сохраните
5. Перейдите на вкладку **Экземпляры**, нажмите **Добавить новый экземпляр**
6. Задайте имя экземпляра, установите статус **Включён**
7. На вкладке **Конфигурация**:
   - Введите VAPID Subject (например, `mailto:admin@example.com`)
   - Включите **Push-уведомления**
   - Включите желаемые типы оповещений
   - При необходимости укажите URL пользовательской иконки уведомления
   - Сохраните — ключи VAPID генерируются автоматически

## Принцип работы

### Конфигурация администратора (Панель администратора > Плагины > Push-уведомления)

| Настройка | Описание |
|---|---|
| Push-уведомления | Главный переключатель вкл/выкл |
| VAPID Subject | Контактный email для идентификации в службе push |
| Ключи VAPID | Генерируются автоматически при первом сохранении |
| Оповещения о новых заявках / сообщениях / назначениях / передачах / просрочках | Глобальные переключатели по типам событий |
| URL иконки уведомления | Пользовательская иконка/логотип для push-уведомлений (оставьте пустым для иконки по умолчанию) |

### Настройки агента (Иконка шестерёнки рядом с колокольчиком в навигационной панели)

Каждый агент может настроить собственные параметры уведомлений:

| Настройка | Описание |
|---|---|
| Переключатели событий | Выбор типов событий, вызывающих push-уведомления |
| Фильтр по отделу | Получать уведомления только из выбранных отделов |
| Режим тишины | Отключать уведомления в заданный диапазон времени (поддерживает ночные промежутки) |

### Поток уведомлений

```
Главный переключатель плагина ВКЛЮЧЁН?
  └─ Переключатель события плагина ВКЛЮЧЁН? (например, alert_new_ticket)
      └─ У агента есть push-подписка?
          └─ Настройка события у агента ВКЛЮЧЕНА?
              └─ Отдел заявки входит в фильтр отделов агента? (пусто = все)
                  └─ Не в режиме тишины агента?
                      └─ ОТПРАВИТЬ PUSH ✓
```

> **Примечание:** Push-уведомления полностью независимы от настроек email-уведомлений osTicket. Вы можете отключить все email-уведомления — push-уведомления продолжат работать.

## Архитектура

| Файл | Назначение |
|---|---|
| `plugin.php` | Манифест плагина (id, version, name) |
| `config.php` | Поля конфигурации администратора + генерация ключей VAPID + создание таблиц БД |
| `class.PushNotificationsPlugin.php` | Bootstrap, обработчики сигналов, маршруты AJAX, инъекция ресурсов |
| `class.PushNotificationsAjax.php` | AJAX-контроллер (подписка, отписка, настройки, тест) |
| `class.PushDispatcher.php` | Отправка уведомлений с логикой получателей и фильтрацией по настройкам |
| `class.WebPush.php` | Чистый PHP Web Push (VAPID + ECDH + AES-128-GCM, без Composer) |
| `assets/push-notifications.js` | Клиентский интерфейс колокольчика/шестерёнки, модальное окно настроек, регистрация Service Worker |
| `assets/push-notifications.css` | Стили для иконок навигационной панели, модального окна, переключателей, тёмной темы |
| `assets/sw.js` | Service Worker для получения и отображения push-уведомлений |

## Таблицы базы данных

Плагин создаёт две таблицы при первом сохранении конфигурации:

- `ost_push_subscription` — хранит конечные точки push-подписок браузера для каждого агента
- `ost_push_preferences` — хранит настройки уведомлений для каждого агента
